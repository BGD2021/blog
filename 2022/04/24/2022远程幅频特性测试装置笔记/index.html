# 题目介绍

# 一、基本要求
  1. 制作一信号源。输出频率范围：1MHz - 40MHz；步进：1MHz，且具有自动扫描功能；负载电阻为 600Ω 时，输出电压峰峰值在 5mV - 100mV 之间可调。
  2. 制作一放大器。要求输入阻抗：600Ω；带宽：1MHz - 40MHz；增益：40dB，要求在 0 - 40 dB 连续可调；负载电阻为 600Ω 时，输出电压峰峰值为 1V，且波形无明显失真。
  3. 制作一用示波器显示的幅频特性测试装置，该幅频特性定义为信号的幅度随频率变化的规律。在此基础上，如图 1 所示，利用导线将信号源、放大器、幅频特性测试装置等三部分联接起来，由幅频特性测试装置完成放大器输出信号的幅频特性测试，并在示波器上显示放大器输出信号的幅频特性。

# 二、发挥部分
  1. 在电源电压为 + 5V 时，要求放大器在负载电阻为 600Ω 时，输出电压有效值为 1V，且波形无明显失真。
  2. 如图 2 所示，将信号源的频率信息、放大器的输出信号利用一条 1.5m 长的双绞线（一根为信号传输线，一根为地线）与幅频特性测试装置联接起来，由幅频特性测试装置完成放大器输出信号的幅频特性测试，并在示波器上显示放大器输出信号的幅频特性。


3. 如图 3 所示，使用 WiFi 路由器自主搭建局域网，将信号源的频率信息、放大器的输出信号信息与笔记本电脑联接起来，由笔记本电脑完成放大器输出信号的幅频特性测试，并以曲线方式显示放大器输出信号的幅频特性。

# 题目分析

# 一、硬件部分
  • 等待硬件队友的帮助
# 二、软件部分
  1. 信号源部分
  • 输出频率上 MHz 首先排除用单片机来产生，必须使用 FPGA 或者带有高速时钟的模块来输出，这里我选用的是 AD9959DDS 模块，模块最高可以产生 200MHz 的正弦波，最高输出幅度为 530mVpp，并且 DAC 输出分辨率为 10 位，完全满足题目的输出频率范围、幅度范围、步进范围，但对于 5mV 这样的小信号来说，DDS 直接输出结果肯定不理想，因此需要硬件来解决这一问题。
  • 题目要求信号源有扫频功能，那么就需要确定扫频的周期，因为硬件方案选用的是检波模块，那么就需要考虑到检波模块的幅度变化的速度是有限的，所以就不能变化的太快，这里初定为 400ms 信号源从 1MHz 以 1Mhz 步进变化到 40MHz。
  1. 测试装置
  • 之前的信号源是作为一个单独部分来作为题目要求，那么测试装置就需要和信号源部分隔离开来，因此需要使用另外一块单片机来实现测试部分，
  • 首先需要用示波器显示出幅频特性曲线，那么就是用示波器的 XY 模式显示，那么需要有一路信号作为触发信号，作为 X 轴来控制光标，那么就需要和扫频周期同步，那么就需要实时接收信号源的频率信息来输出幅度按照频率变化的信号来作为触发信号。题目中要求使用有线信道进行通信，单片机之前提到有线信道，那么便就是串口了。
  • 题目要求要使用 WIFI 自主搭建局域网将幅频特性曲线上传到上位机进行显示，这里就可以用到 WIFI 模块 ESP8266 来实现，可以使用串口和 AT 指令来连接局域网并与上位机进行通信。能与上位机通信后，还需要考虑上位机处理数据和显示幅频特性曲线的问题，受到 9cats せんせい的启发，有一个支持 html5 的一个图标图形库，自带自动对曲线拉伸调节并且抗锯齿，实现效果非常不错，那么就需要解决 ESP8266 与网页进行通信的问题，html 使用的是 http 的协议进行通信，ESP8266 是通过 TCP/IP 的协议与上位机进行通信，那么就想到使用 node 创建 HTTP 和 TCP 两个服务端来进行转发，但可惜这里我水平有限没有办法说得很详细，node 中有 HTTP 和 TCP 的模块，监听本机的两个端口，一个 TCPServer 用来接收 8266 发送的数据，HTTPServer 再将 TCPServer 接收到的数据发送到网页监听的端口，再使用图形库进行显示。

# 遇到的问题

# 1. 扫频问题

如果使用 1MHz-40MHz 再返回到 1MHz-40MHz 的扫频方案，那么有可能会由于检波幅度变化需要时间出现在 40MHz 变为 1MHz 的时候出现一个幅度逐渐增大的曲线的问题，而这个曲线不属于被测网络的幅频特性，便就出现了错误，解决这个问题有两个办法:

一个是扫频在扫完一个周期时，停止一段时间留给硬件电路复位，同时也为软件留出了数据处理的时间。

第二个方案也是我们使用的方案，将扫频变成 1MHz-40Mhz-1MHz，那么就可以解决这个问题，同时不影响程序流程，将触发信号输出锯齿波变成输出三角波。

# 2.DDS 输出幅度问题

在我们使用 DDS 扫频进行测试的过程中在示波器上发现 DDS 输出不同频率的正弦波会出现幅度不等的问题，刚开始我们怀疑问题出现在后面的硬件电路上，有可能出来的信号带有衰减再放大的硬件的幅频特性，后来经分析和测试发现排除这个可能性，首先衰减使用的是商用的物理衰减头，同时放大器 OPA690 芯片的带宽远超 40MHz，后来发现问题原因在于 DDS 模块中集成了一个 200MHz 的低通滤波器，因此我们从软件上解决了这个问题：
