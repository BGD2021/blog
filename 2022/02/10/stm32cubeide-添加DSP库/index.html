# 一、前言

在嵌入式的学习过程中会碰到各种各样的数据处理要求，如果使用单片机利用算法强行计算效率就太低并且效果并不理想，因此我们会需要用到 DSP 库来处理数据，对于有 FPU（浮点运算单元）的单片机计算性能大幅提高，对大量数据的处理效果更为显著，并且 DSP 库中不仅有数学函数，还有很多信号处理的算法，如 FFT，FIR，IIR 等，因此移植 DSP 还是非常有必要的。

stm32cubeide 是 st 开发的一个集合了 stm32cubemx、编译环境的一个一体式多操作系统开发工具，具有 STM32 微控制器和微处理器的外设配置、代码生成、代码编译和调试功能。它基于 Eclipse®/CDT™框架和用于开发的 GCC 工具链，以及用于调试的 GDB。它支持集成数以百计的现有插件，因其便利的调试功能、更快的编译速度、当然还有更舒服的界面，成了我目前的首选开发环境。

# 二、移植

# 1. 添加 DSP 库

  • 首先在 stm32 的库中找到 DSP 库的头文件、源文件、静态库，在我的电脑中路径是 C:\Users\lenovo\STM32Cube\Repository\STM32Cube_FW_F4_V1.26.2\Drivers\CMSIS
  • 打包成这样的一个文件夹，放入到 ide 项目中的 driver 文件夹中

# 2. 引入头文件

# 3. 添加宏定义

__FPU_PRESENT，数值 1

__FPU_USED，数值 1

__TARGET_FPU_VFP

ARM_MATH_CM4

# 4. 添加静态库文件

  • 在 setting 中找到 MCU GCC Linker，在找到其中的 miscellaneous 项，添加（双引号不要漏）
    "{workspace_loc:/{ProjName}/Drivers/DSP/libarm_cortexM4lf_math.a}"

# 5. 完成

在项目中引入
#include "arm_math.h"

再编译工程
