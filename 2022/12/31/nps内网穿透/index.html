# NPS 介绍

nps 是一款轻量级、高性能、功能强大的内网穿透代理服务器。目前支持 TCP、UDP 流量转发，可支持任何 tcp、udp 上层协议（访问内网网站、本地支付接口调试、ssh 访问、远程桌面，内网 dns 解析、游戏私服、我的世界服务器等等……），此外还支持内网 http 代理、内网 socks5 代理、p2p 等，同时支持 snappy 压缩、站点保护、加密传输、多路复用、header 修改等。支持 web 图形化管理，集成多用户模式。

比如

  • 做微信公众号开发、小程序开发等 ----> 域名代理模式
  • 想在外网通过 ssh 连接内网的机器，做云服务器到内网服务器端口的映射，----> tcp 代理模式
  • 在非内网环境下使用内网 dns，或者需要通过 udp 访问内网机器等 ----> udp 代理模式
  • 在外网使用 HTTP 代理访问内网站点 ----> http 代理模式
  • 搭建一个内网穿透 ss，在外网如同使用内网 vpn 一样访问内网资源或者设备 ----> socks5 代理模式
    NPS 项目 Github 地址：https://github.com/ehang-io/nps

# 前期准备

一台带有公网 IP 的云服务器或者是 VPS，内网穿透过去的带宽取决于你服务器的带宽，所以最好选择一台带宽大一点的服务器。

# 搭建过程

  1. 下载 nps

    下载地址：https://github.com/ehang-io/nps/releases

    我的华为云服务器是 ubuntu 的，就下载 linux_amd64_server.tar.gz 了

    再通过 scp 发送到服务器上
    scp linux_amd64_server.tar.gz user@ip:./folder

  2. 安装 nps server

    解压文件
    tar zxvf linux_amd64_server.tar.gz -C /opt/programs

    在服务器上解压过后会有这几个文件

    进入 conf 文件夹，打开编辑 nps.conf 文件

    web_host 修改为你的 ip，
    nps 默认配置文件使用了 80，443，8080，8024 端口

    80 与 443 端口为域名解析模式默认端口

    8080 为 web 管理访问端口

    8024 为网桥端口，用于客户端与服务器通信

    如果有服务部署在 80 端口可以把他放在其他端口，服务器安全组记得开放这些端口

    8080 是 web 端后台的端口，username 和 password 就是登录后台的账号密码

    安装命令
    sudo ./nps install
    sudo nps start
    此时访问服务器的 8080 端口，出现以下页面就成功

  3. 安装 nps client

    登录到后台后，新建一个客户端，压缩和加密都选 “是”

    再到 nps 下载地址中下载一个客户端

    解压后打开 conf 文件夹中的 npc.conf 配置文件，把 server_addr 改成云服务器 ip:8024，vkey 改成刚刚新增客户端的唯一验证密钥，如图所示，后面的内容可以都删掉
